## Web-презентация из 5 страниц в стилистике флагманского бутика BORK

## Роль

Ты — senior web designer, арт-директор и front-end developer, специализирующийся на премиальных digital experience, motion design и интерактивных web-презентациях.

Твоя задача — создать **лендинг-презентацию из 5 полноэкранных страниц/разделов** в эстетике флагманского бутика BORK.

Это должен быть не классический e-commerce лендинг и не набор отдельных экранов, а **единое кинематографичное digital-пространство**, которое можно использовать вместо презентации на встрече, защите идеи, внутреннем выступлении или демонстрации концепции.

Пользователь должен ощущать, что он не скроллит сайт, а **проходит через пять последовательных залов флагманского бутика BORK**.

# 2. Формат: 5 полноэкранных страниц

Лендинг должен состоять ровно из **5 ключевых страниц/экранов**.

Каждая страница:

* занимает минимум `100vh`;
* работает как отдельный слайд презентации;
* имеет собственную смысловую роль;
* визуально связана с предыдущей и следующей страницей;
* плавно переходит в следующую через motion design;
* не должна выглядеть как самостоятельный несвязанный блок.

Навигация может быть реализована через:

* vertical scroll;
* snap-scroll;
* controlled scroll presentation;
* keyboard navigation;
* боковой progress indicator;
* тонкую нижнюю timeline-линейку;
* номера страниц `01 / 05`, `02 / 05` и т.д.

Рекомендуемая логика:

```css
html {
  scroll-behavior: smooth;
}

.presentation-section {
  min-height: 100vh;
  scroll-snap-align: start;
}
```

## Страница 1 — Entrance / Вход в пространство BORK

### Задача

Мгновенно погрузить пользователя в атмосферу флагманского бутика.

### Визуальное решение

* fullscreen hero;
* крупное интерьерное изображение;
* центральная перспектива;
* темный теплый overlay;
* мягкое движение камеры;
* вертикальные архитектурные линии;
* тонкий progress indicator `01 / 05`.

### Текст

Заголовок:

**BORK**
**Пространство современной эстетичной жизни**

Подзаголовок:

Техника и аксессуары на стыке искусства, инженерной точности и инноваций.

CTA:

**Исследовать пространство**

### Motion

При загрузке:

1. фон появляется через slow scale `1.06 → 1.00`;
2. затем проявляются вертикальные рейки;
3. проходит тонкая теплая световая линия;
4. заголовок появляется через mask reveal;
5. подзаголовок и CTA появляются с задержкой.

## Страница 2 — Philosophy / Новое понимание роскоши

### Задача

Показать философию BORK: роскошь не через демонстративность, а через тишину, точность и внимание к деталям.

### Визуальное решение

* warm ivory / stone фон;
* крупная типографика;
* много воздуха;
* 3 философских принципа;
* тонкие линии;
* легкая текстура бумаги или камня.

### Смысловые блоки

1. **Сдержанность**
   Роскошь без визуального шума.

2. **Точность**
   Каждая линия, материал и сценарий имеют значение.

3. **Совершенство**
   Пространство, где технология становится частью эстетики.

### Motion-переход со страницы 1

Переход должен напоминать вход из темного бутика в светлое архитектурное пространство:

* темный фон страницы 1 раскрывается через вертикальную маску;
* теплый свет проходит слева направо;
* ivory-фон страницы 2 проявляется постепенно;
* текст появляется не сразу, а после завершения раскрытия пространства.

# 8. Motion design: ключевые принципы

Motion design должен быть одним из главных элементов проекта.

## Общий характер движения

Использовать:

* slow reveal;
* parallax;
* controlled scroll;
* mask transition;
* vertical shutter transition;
* light reveal;
* scale transition;
* depth layering;
* smooth typography reveal;
* scroll-triggered animations;
* плавную смену фонов;
* задержки между появлением изображения и текста.

Не использовать:

* bounce;
* elastic;
* случайные fly-in;
* rotate;
* резкие wipe;
* слишком быстрые появления;
* шаблонные PowerPoint-эффекты;
* агрессивные hover-анимации.

## Easing

```css
:root {
  --ease-premium: cubic-bezier(0.22, 1, 0.36, 1);
  --ease-soft: cubic-bezier(0.16, 1, 0.3, 1);
  --ease-cinematic: cubic-bezier(0.19, 1, 0.22, 1);
}
```

## Базовая анимация появления

```css
.reveal {
  opacity: 0;
  transform: translateY(32px) scale(0.98);
  transition:
    opacity 900ms var(--ease-premium),
    transform 1100ms var(--ease-premium);
}

.reveal.active {
  opacity: 1;
  transform: translateY(0) scale(1);
}
```

## Фирменный переход между страницами

Реализовать **architectural shutter transition**:

1. при переходе появляются вертикальные графитовые рейки;
2. между ними проходит теплая световая линия;
3. новая страница раскрывается через маску;
4. фон появляется первым;
5. затем проявляются архитектурные линии;
6. после паузы 200–350 мс появляется текст;
7. в финале остается небольшой акцентный элемент: линия, номер страницы или маркер.

Переход должен напоминать не смену слайда, а **прохождение через архитектурный портал**.

## Scroll behavior

Реализовать один из вариантов:

### Вариант 1 — Premium snap-scroll

* каждая секция занимает `100vh`;
* скролл фиксируется на секциях;
* переходы запускаются при входе секции в viewport;
* user сохраняет естественное управление.

### Вариант 2 — Controlled presentation mode

* колесо мыши или клавиатура переключают страницы;
* переходы более кинематографичные;
* подходит для использования как презентации.

Если проект должен использоваться именно вместо презентации, предпочтительнее **controlled presentation mode** с поддержкой:

* mouse wheel;
* keyboard arrows;
* touch swipe;
* progress indicator;
* возможность перейти к любой из 5 страниц.

# 13. Mobile-адаптация

На mobile лендинг должен оставаться презентационным и премиальным.

Правила:

* все 5 страниц сохраняются;
* каждая страница остается отдельным смысловым экраном;
* заголовки уменьшаются, но сохраняют выразительность;
* анимации упрощаются;
* heavy parallax отключается;
* vertical shutter можно заменить на light fade + mask reveal;
* CTA остается видимым;
* изображения кадрируются по смысловому центру;
* текст не должен выходить за границы.

Обязательно учитывать:

```css
@media (prefers-reduced-motion: reduce) {
  * {
    animation: none !important;
    transition: none !important;
    scroll-behavior: auto !important;
  }
}
```

# 14. Что нельзя делать

Категорически избегать:

* обычного лендинга из независимых блоков;
* ощущения шаблонной презентации;
* резких переходов между страницами;
* случайных анимаций;
* bounce / elastic / fly-in;
* визуального шума;
* маркетплейсной эстетики;
* карточек товаров как в интернет-магазине;
* скидочных элементов;
* ярко-золотых градиентов;
* кислотных цветов;
* cold blue glow;
* cyberpunk;
* огромных скруглений;
* мультяшных иллюстраций;
* перегруженных dashboard-графиков;
* агрессивных CTA;
* слишком большого количества текста на одном экране.

# 15. Критерии готовности

Результат считается успешным, если:

1. Создана web-презентация ровно из 5 полноэкранных страниц.
2. Все 5 страниц выглядят как части одного цельного маршрута.
3. Переходы между страницами плавные, современные и кинематографичные.
4. Motion design является частью концепции, а не декоративным эффектом.
5. Визуальный стиль соответствует флагманскому бутику BORK.
6. Есть ощущение архитектурного пространства, глубины и материальности.
7. Цвета теплые, глубокие и премиальные.
8. Оранжевый акцент используется точечно.
9. Сайт можно использовать как презентацию.
10. На desktop проект выглядит как premium brand experience.
11. На mobile сохраняется читаемость, эстетика и скорость.
12. Код структурирован, адаптивен и готов к развитию.
