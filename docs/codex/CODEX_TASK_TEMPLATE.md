# CODEX_TASK_TEMPLATE.md

Use this template for future Codex tasks.

```text
Задача:
<кратко и конкретно, что нужно сделать>

Контекст:
<почему это нужно и к какой части презентации относится>

Ограничения:
- не менять количество слайдов, если это не является целью задачи;
- не переписывать проект на другой стек;
- не трогать loader/navigation/mobile Safari без необходимости;
- не использовать `git add .`;
- не коммитить экспериментальные ассеты случайно.

Разрешенные файлы:
- <например: index.html>
- <например: styles.css>
- <например: script.js>

Нельзя менять:
- <перечислить файлы или зоны, которые нельзя трогать>

Критерий готовности:
- <измеримый критерий 1>
- <измеримый критерий 2>
- <визуальная/QA проверка>

Перед началом:
1. Проверь `git status --short`.
2. Назови точные файлы, которые планируешь изменить.
3. Дай короткий план.
4. Не вноси изменения, пока план не будет подтвержден, если задача не является очевидной микроправкой.

После изменений:
1. Покажи краткий summary.
2. Покажи команды проверки.
3. Укажи риски и что нужно проверить вручную.
```

## Example: safe micro-task

```text
Задача:
Добавить короткий документированный smoke-check для проверки структуры презентации.

Ограничения:
- не менять дизайн;
- не менять HTML-структуру слайдов;
- не менять loader/navigation;
- не добавлять тяжелые зависимости.

Критерий готовности:
- проверяется, что `.PresentationSection` ровно 5;
- проверяются `data-index` 0..4;
- проверяется наличие 5 progress dots;
- проверяется наличие кнопки `Вернуться в начало`;
- есть команда запуска проверки.
```
